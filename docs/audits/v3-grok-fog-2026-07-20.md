# Village Siege v0.8.0 fog-of-war audit

Date: 2026-07-20

Scope: offline deterministic fog-of-war vertical slice

Lead: Codex

Independent supervisor: Grok CLI (read-only)

## Verdict

Accepted for the v0.8.0 offline slice. Grok reported no P0 or P1 findings after the final fixes. The mobile/static and projectile/rules reviewers independently reached the same P0=0, P1=0 result for this scope.

This approval does not cover complete RTS, fortifications or authoritative multiplayer. The browser-local runtime remains the single-player authority and therefore is not an online secrecy or anti-cheat boundary. Server-owned simulation and recipient-only transport remain TASK-018 and later release gates.

## Audited behavior

- Deterministic per-player current and explored tiles with team-shared vision.
- Hostile moving units disappear outside vision; hostile buildings retain copied, non-interactive last-sighting views.
- Re-scouting any empty remembered footprint cell clears the stale building.
- Recipient snapshots and same-tick event frames filter hidden entities, projectiles, target points, target IDs, damage sources, removals and resource lifecycle events.
- Commands cannot directly target hidden entities; construction requires a fully visible footprint; movement into fog cannot probe hidden occupancy during validation.
- AI observations and scouting memory derive from authoritative visibility rather than injected hidden coordinates.
- Snapshot-driven projectiles move through visibility transitions; ordinary arrows use a launch-time terrain-checked ballistic target cell and can miss moving targets.
- The mobile contextual dock remains seven controls and fits the 568x320 landscape gate without overlap.

## Verification evidence

- `npm run verify`: passed (typecheck, tests and workspace builds).
- Shared tests after the final ballistic regression: 140/140 passed.
- Targeted simulation plus visibility suite: 86/86 passed.
- Client production build: passed; the only warning is the existing large JavaScript chunk advisory.
- Local multiplayer lobby/reconnect smoke: PASS with two players, host/readiness/payload rejection checks and authoritative lobby tick.
- `npm audit --omit=dev`: zero vulnerabilities.
- `git diff --check`: passed.
- Playwright 568x320: document 568x320, canvas 568x319, exactly seven visible controls, zero console errors; 667x375 and 844x390 were also checked during the slice.

## Original design and licensing

The slice uses original Village Siege names, presentation, rules expression and balance. It does not copy Age of Empires artwork, audio, factions, maps, interface composition, source code or proprietary values. "Private design" means original ownership and identity; the reusable client, server, shared rules, art sources and deployment material remain MIT licensed and self-hostable.

## Explicit remaining scope

- Walls, gates, landmarks and rubble path transitions are not implemented.
- The current multiplayer service is a lobby/reconnect prototype, not authoritative online combat.
- Online clients do not yet receive only `VisibleSnapshot` and projected events from a server-owned match.
- Save/replay recovery, latency/packet-loss gates and five-player authoritative load tests remain pending.
