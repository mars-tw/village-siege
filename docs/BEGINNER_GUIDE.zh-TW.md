# Village Siege 新手指南

這份指南寫給第一次接觸 Village Siege 的玩家與開發者。只想玩單機時，不需要帳號、API 金鑰或多人伺服器。

只想直接遊玩，請開啟 **[GitHub Pages 公開單機版](https://mars-tw.github.io/village-siege/)**；想研究、修改或回報問題可前往 **[GitHub 原始碼](https://github.com/mars-tw/village-siege)**。後面的本機安裝步驟主要提供給開發者。

## 一、五分鐘開始單機遊戲

### 1. 準備環境

你需要：

- Windows 10／11、macOS 或常見 Linux 發行版。
- Node.js 22.12 或更新版本。
- npm 11 或相容版本；安裝 Node.js 時通常會一併安裝。
- Chrome、Edge 或 Firefox 等支援 WebGL 的現代瀏覽器。

先確認版本：

```powershell
node --version
npm --version
```

如果 `node` 顯示的版本低於 `v22.12.0`，請先更新 Node.js。

### 2. 下載專案

已安裝 Git 時：

```powershell
git clone https://github.com/mars-tw/village-siege.git
Set-Location village-siege
```

也可以在 GitHub 專案頁選擇 **Code → Download ZIP**，解壓縮後用 PowerShell 進入該資料夾。

### 3. 安裝並啟動

在專案根目錄執行：

```powershell
npm ci
npm run dev:client
```

終端機出現網址後，用瀏覽器開啟 `http://localhost:5173`。如果 5173 已被其他程式使用，Vite 可能顯示另一個連接埠，請以終端機實際網址為準。

> 不要直接雙擊 `apps/client/index.html`；遊戲需要透過 Vite 啟動，才能正確載入模組與素材。

## 二、第一場戰役怎麼玩

第一次建議選擇：

- 村莊：**松林堡**，定位較均衡，適合熟悉操作。
- 電腦對手：**均衡者**，壓力比侵略者容易掌握。
- 模式：**開始單機戰役**。

進入戰場後按這個順序：

1. 按底部 **全選工匠**，再點附近的木材、糧食或石礦；上方資源數字會增加，資源點數量會下降。
2. 選取工匠後按 **建造**，先蓋一棟 **拓荒家屋**，完工後人口上限會由 10 增加到 18。
3. 再蓋 **邊軍兵營**。放置時要替整片 2×2 基地留空；工匠會走到工地，建築會從地基、鷹架逐步完成。
4. 點完成的兵營先訓練戰士與槍衛；接著蓋 **射箭庭** 訓練弓手。法師、火銃兵、野豬斥候與重弩攻城組也各有自己的專屬生產建築。
5. 產出至少三名軍事單位後按 **全選軍隊**，用 **攻擊移動** 往東探索；深色區域從未偵察，較淡的陰影代表曾探索但目前沒有我方視野。接近敵人後點可見敵軍或敵方建築進攻。
6. 留意上方勝途列；你可以攻陷敵方議事核心，也可以改走殲滅、拓界標或中域控制，不必每場都用同一種打法。

木作營與糧秣所不是裝飾：在對應資源點六格內完工後，附近木材或糧食的採集量會提高 50%。守望塔會自動攻擊進入射程的敵軍。

敵方移動單位離開視野後會消失，不能再用舊位置直接點名攻擊；請對最後方向使用攻擊移動。已偵察過的敵方建築會在地圖留下不可選取的半透明「最後目擊」殘影，但隱藏期間不會偷更新生命值；再次偵察空地後，過期殘影會被清除。

### 四條勝途

- **議事核心征服**：敵方最後一座完整議事核心被摧毀後開始 60 秒重建寬限；期限內完成重建可解除倒數，只有地基不算。
- **全軍殲滅**：讓敵方失去所有有效單位與戰略建築。孤立工地、城牆、城門、殘骸或飛行中的箭矢不能單獨保住隊伍。
- **拓界標控制**：完成至少一座拓界銅標並連續持有 90 秒；標誌被摧毀或不足時進度歸零，多蓋不會加速。
- **中域控制**：開局 60 秒後，中域開始計分；只有我方活體單位在圈內時每個模擬 tick 累積一次，敵我同時進圈會暫停，累積 120 秒獲勝。

同一 tick 若不同隊伍同時達成勝途，結果會明示為和局。戰局結束後，上方結果會持續顯示，底部保留 **再戰**、**下載重播** 與 **返回**。

## 三、v0.9 建築、產兵與築城分工

Village Siege 不讓一棟兵營生產所有軍隊。每座軍事建築與城防都有自己的戰場剪影、成本、施工時間與規則職責：

| 建築 | 占地 | 生產／作用 | 外觀辨識 |
|---|---:|---|---|
| 拓荒家屋 | 1×1 | 增加 8 人口上限 | 小型木石家屋 |
| 木作營／糧秣所 | 2×1 | 提高鄰近木材／糧食採集 | 原木架／糧袋農具 |
| 邊軍兵營 | 2×2 | 戰士、持盾槍衛 | 軍旗與交叉兵器 |
| 守望塔 | 1×1 | 自動遠程防禦 | 高聳石塔 |
| 射箭庭 | 2×2 | 弓手 | 靶場與箭架 |
| 星火院 | 2×2 | 法師 | 氧化銅天文環與發光爐 |
| 火器坊 | 2×2 | 火銃兵 | 排煙筒與火藥桶 |
| 獠騎圈 | 2×2 | 野豬斥候 | 圍欄、獸棚與獠獸 |
| 攻城棚 | 3×2 | 重弩攻城組 | 大型木架、輪軸與雙輪 |
| 樹脂石籠牆 | 1×1 | 封鎖入口、承受攻擊 | 樹脂綁木與編織頁岩石籠 |
| 測界雙葉門 | 2×1／1×2 | 可開關的部隊通道 | 不對稱門葉與測界牌 |
| 拓界銅標 | 2×2 | 城寨期戰略地標 | 氧化銅圓盤與測量桿 |

放置建築時，綠色預覽代表整片占地都可使用；紅色代表至少一格超出地圖、碰到地形、資源或其他建築。測界雙葉門可在放置列切換北東／南東軸；完工後點門，再用第一格開啟或關閉。關門會封鎖單位路線，開門可讓部隊穿越；門格內有人時不能關閉。牆或門被摧毀會留下「可通行破口」，部隊可穿越，但殘骸消散前不能在同一格重新施工。攻城棚最寬，建議放在基地外緣並保留出口。

## 四、操作表

| 操作 | 功能 |
|---|---|
| 點／左鍵我方單位或建築 | 選取；兵營與議事堂會自動顯示可訓練項目 |
| `Shift` + 左鍵我方單位 | 加入或移出目前選取 |
| 點／左鍵資源 | 已選工匠前往並持續採集 |
| 點／左鍵空地 | 已選單位移動；建造模式時放置建築 |
| 點／左鍵敵人 | 已選軍事單位指定攻擊目標 |
| 拖曳空地 | 平移戰場鏡頭；拖曳不會誤放建築 |
| `B` | 以目前選取的工匠開啟四頁建造選單 |
| `WASD` | 平移戰場鏡頭 |
| 滑鼠滾輪／Canvas `＋` `－` | 拉近或拉遠鏡頭 |
| `P` | 暫停或繼續共享模擬 |
| `R` | 重新開始目前戰役 |
| `Esc` | 依序取消戰術選點、取消確認、佇列／集結、建造；沒有子模式時返回戰前會議 |

手機橫向時使用完全相同的智慧點擊。底部只有一排情境指揮塢：選取工匠後按 **建造**，再用 **下一頁／回首頁** 循環瀏覽四頁建築，選好後點戰場放置；放置城門時同一列可旋轉，點完成城門時同一列可開關。選取各軍事建築則只顯示它能生產的兵種。568×320 下操作列不會和上方資源列重疊；直向時遊戲會暫停並顯示橫放提示。

### 存檔與重播

在底部按 **系統 → 存檔重播**，同一列固定七格可匯出／匯入存檔與重播，也能單獨匯出操作日誌。匯入會先在隔離的候選戰局中檢查 JSON 結構、檔案大小、schema、protocol、rules 版本、AI／實體完整結構與逐操作 hash；全部通過後才替換畫面中的戰局，失敗不會破壞目前進度。存檔會保留被拒操作已消耗的玩家序號、未滿一個模擬 tick 的時間餘數與 AI budget，並以 continuation hash 綁定最終狀態，讀檔後不會重用工作 ID 或改變下一步時序。

這些 `.vssave`／`.vsreplay` 是擁有者本機使用的完整權威資料，包含尚未探索的敵軍、AI 私有記憶與規劃，不是適合公開分享給對手的去霧版本。檔內 FNV hash 只能找出意外損壞或重播不同步，不是簽章、加密或惡意竄改防護。

## 五、七種軍事單位快速認識

| 兵種 | 生產建築 | 主要用途 | 新手提示 |
|---|---|---|---|
| 戰士 | 邊軍兵營 | 低成本持續近戰 | 最快組成第一波防線 |
| 持盾槍衛 | 邊軍兵營 | 兩格攻距與較高單次傷害 | 站在弓手前方接敵 |
| 弓手 | 射箭庭 | 五格遠程輸出 | 保持距離，避免被斥候貼身 |
| 法師 | 星火院 | 高傷害中距輸出、人口 2 | 石材成本高，適合補強成熟經濟 |
| 火銃兵 | 火器坊 | 高單發遠程、人口 2 | 攻擊較慢，需要前排保護 |
| 野豬斥候 | 獠騎圈 | 最高移速與大視野 | 先偵查敵城，再帶主力進攻 |
| 重弩攻城組 | 攻城棚 | 高生命與高建築傷害、人口 3 | 木石成本高，適合終結敵方議事堂 |

工匠不列入七種軍事單位；它負責採集與施工。沒有任何單一兵種能處理所有情況，新手最穩定的做法是「戰士／槍衛在前、弓手／法師／火銃在後、野豬斥候先探路、重弩攻城組拆主城」。

## 六、多人房間

目前多人功能包含 2～4 人房間、六碼房號、準備、房主開始、server-owned lobby clock 與 60 秒重連。這個 clock 尚未執行共享戰場模擬；**戰場的完整多人同步尚未完成，因此目前是連線架構原型，不是完整線上對戰版本。**

本機測試需要兩個終端機。

終端機一：

```powershell
npm run dev:server
```

終端機二：

```powershell
$env:VITE_COLYSEUS_URL = "http://127.0.0.1:2567"
npm run dev:client
```

接著：

1. 第一個瀏覽器選擇「多人連線」並建立房間。
2. 記下畫面上的六碼房號。
3. 用第二個瀏覽器或無痕視窗進入同一網址，輸入房號加入。
4. 所有玩家按下「準備」。
5. 房主按下「開始戰局」。

## 七、常見問題

### 第一次進入戰役為什麼會顯示載入中

v2 使用 lazy-loading：公開首頁與選村畫面不會預先下載所有角色動作表，進入單人戰役後才載入戰場所需素材。第一次開啟可能短暫顯示載入狀態；同一瀏覽器再次進入時通常可使用快取。若長時間沒有完成，請檢查網路與瀏覽器主控台的第一個素材錯誤。

### `node` 或 `npm` 不是可辨識的命令

Node.js 尚未安裝，或安裝後終端機沒有重新開啟。安裝 Node.js 22.12 以上版本，再關閉並重新開啟 PowerShell。

### `npm ci` 失敗

請確認目前位置是含有根目錄 `package.json` 與 `package-lock.json` 的 Village Siege 資料夾，並確認網路可連到 npm registry。不要在 `apps/client` 子目錄執行根目錄安裝指令。

### 瀏覽器只有空白畫面或素材載入失敗

1. 確認是使用 `npm run dev:client` 顯示的網址，而不是直接開啟 HTML。
2. 回到終端機查看第一個錯誤訊息。
3. 停止伺服器後重新執行 `npm ci` 與 `npm run dev:client`。
4. 確認瀏覽器已開啟硬體加速並支援 WebGL。

### 連不上多人房間

- 確認 `npm run dev:server` 仍在執行，且顯示監聽 `http://localhost:2567`。
- 確認客戶端啟動前已設定正確的 `VITE_COLYSEUS_URL`。
- 若改用其他連接埠，伺服器與客戶端設定必須一致。
- Windows 防火牆詢問 Node.js 網路權限時，至少允許私人網路。

改用 2667 連接埠的範例：

```powershell
# 終端機一
$env:PORT = "2667"
npm run dev:server

# 終端機二
$env:VITE_COLYSEUS_URL = "http://127.0.0.1:2667"
npm run dev:client
```

### 如何確認下載的版本可以正常建置

```powershell
npm run verify
```

這會執行型別檢查、共享模擬測試與正式建置。多人房間另可執行：

```powershell
npm run smoke:multiplayer:local
```

## 八、參與開源開發

Village Siege 使用 [MIT License](../LICENSE)。歡迎 fork、研究、修改與提交 pull request。

提交前請：

1. 從最新 `main` 建立自己的功能分支。
2. 不要提交 `.env`、`node_modules`、`dist` 或個人登入資料。
3. 執行 `npm run verify`。
4. 若修改多人功能，再執行 `npm run smoke:multiplayer:local`。
5. 在 pull request 說明改了什麼、原因、玩家影響與驗證結果。

美術素材與產生流程的授權說明請參考 [素材署名與來源](../assets/ATTRIBUTION.md)。
