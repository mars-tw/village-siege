# Village Siege 版本紀錄

公開遊戲：[mars-tw.github.io/village-siege](https://mars-tw.github.io/village-siege/)

## v3 v0.19（六向弓箭手、友軍動態避讓，2026-07-22）

- 發行三元組為 app `0.19.0`、protocol `village-siege-network/4`、rules `village-siege/0.18.0`；production Compose、環境範例與 `/version` 驗收文件使用相同版本。
- v0.19.0 將弓箭手升級為六個獨立方向、每方向六動作四影格的原創透明動作表。144 個弓箭手 runtime cells 已通過空格、重複格、跨方向原圖／鏡像重用、透明 RGB 與安全邊界檢查；Chromium 另完成 36/36 方向／動作切換、六張 HTTP 解碼與零載入錯誤門檻。戰士與弓箭手目前都不靠水平翻轉切換方向。
- 共享決定論模擬新增同隊單位逐 tick 格位預約、固定 ID 順序與有界局部繞行。一般移動、巡邏、攻擊接近、施工／採集接近、野豬衝鋒、推擠、燃燒／戰鬥死亡與同隊跨玩家單位都不會搶占同一格；靜態封路保留移動進度，動態捷徑被友軍占用時不再在兩格間反覆折返。40 單位雙格門在 600 tick 內完成通行，shared simulation 89/89 與五種 AI 的 18,000 tick 長跑均通過。
- 844×390 Chromium 從首頁真實點入 `VillageAssaultScene`，分別驗證預設戰場與由「全選工匠 → 建造」開啟的建造選單；兩種狀態各七個可見操作鍵，重疊 0、越界 0、頁面／請求／素材錯誤 0。
- GitHub Pages 建置可由 repository variables 注入正式多人 runtime config，但預設仍明確關閉；只有完成長期公開 WSS 與雙客戶端 live gate 後才可設定 `VILLAGE_SIEGE_MULTIPLAYER_ENABLED=true`。原始碼、美術來源、自架、TLS/WSS、監控與備份範本維持 MIT 開放。

## v3 v0.18（單機公開、權威多人主機待部署，2026-07-21）

- v0.18.0／TASK-022 將協定升為 `village-siege-network/4`、規則升為 `village-siege/0.17.0`。多人房主可加入最多三個伺服器權威 AI，與真人合計最多五個陣營；AI 不占 WebSocket 席位、不接收 hello／frame／ack，決策、命令、序號、恢復與 canonical hash 全留在伺服器。
- 真實大廳 smoke 已完成 2 真人＋3 AI 的五陣營交接。600 tick 五陣營 soak 證明三個 AI 的自發命令零拒絕、雙真人 filtered snapshot／delta 全程可重建、兩次恢復產生相同最終 canonical hash；被竄改的 AI journal 會 fail-stop。
- 逆境雙客戶端 smoke 在 Colyseus frame 抵達後，對每端精確取樣 50 個 delta、丟 1 個（2%），以 50／100／200 ms 決定論收件延遲製造亂序，通過斷鏈偵測、完整 Snapshot resync、真實 socket drop、`recovering → resumed` 與兩端最終可見 hash 收斂。這不是 proxy／netem 傳輸整形；真實 transport 證據僅涵蓋 socket drop／reconnect。另新增越權、霧區、ownerControl、ack、跨收件者 delta 與 AI authority rollback 邊界測試。
- 配對 HTTP 與 WebSocket 握手改用共用 exact-origin 白名單，將 Colyseus 預設的 credentialed wildcard CORS 改成 credentialed exact-origin（瀏覽器 SDK 的配對 fetch 需要 credentials），限制公開方法與連線負載／逾時；靜態客戶端新增不允許 inline script／eval 的 CSP。自有部署可用 `ALLOWED_ORIGINS` 明確加入客戶端 origin。
- v0.18 原始碼、GitHub Pages 單機版、Docker／production Compose、TLS/WSS 邊界、監控與備份範本已通過 CI。正式 Release 會發布可匿名拉取的 `linux/amd64`／`linux/arm64` GHCR client/server 映像、SBOM 與 provenance attestation；client 以 `PUBLIC_CONNECT_ORIGIN` 啟動時注入 runtime config，同一 digest 不綁網域。長期公開 WSS 主機與真實公開雙客戶端終驗仍屬 TASK-026，因此尚未宣稱公開多人版已上線。

- v0.17.0／TASK-021 將協定升為 `village-siege-network/3`、規則升為 `village-siege/0.16.0`。玩家專屬 Snapshot 新增安全的參戰者、聚落進度、己方生產控制與活動提示；生產佇列、AI authority、霧外目標與其他玩家私有資料仍不外洩。
- `MultiplayerLobbyScene` 在第一個已驗證 frame 只交接一次，並沿用同一個 `MultiplayerClient` 啟動 `VillageAssaultScene`。線上場景使用獨立來源與 10 Hz 有界位置插值，不呼叫單機 `runtime.step()`，所有採集、建造、產兵、研究、戰術、技能與投降都只送 command intent，等待伺服器回執與 frame。
- 修正 Colyseus 初始 roster 尚未實體化時的瀏覽器競態、素材失敗返回時的殘留連線，以及高負載下過短的 authority fencing lease。568×320 橫向畫面可完整容納七格主／子指令列；多指操作由單一 pointer 擁有，第二指不會誤觸下令。
- 真實 Chromium 雙瀏覽器已完成不同村莊建房、加入、雙方準備、啟動權威戰場與採集命令回執；TASK-022 的延遲、丟包、惡意命令、五方＋AI 與最終 hash 閘門仍未完成，因此 v0.17 尚未公開部署。

- v0.16.0／TASK-020 將網路協定升為 `village-siege-network/2`、規則升為 `village-siege/0.15.0`；新增 120 秒不續延 reconnect lease、穩定 logical match ID、`recovering/resumed/failed` lifecycle，以及 full Snapshot 後才解鎖的 pending intent 有序重播。
- 權威 recovery record 每 20 tick checkpoint，保存短 batch-tick journal、序號游標、reorder buffer 與 immutable accepted/rejected result ledger；任何 hash、版本、指令 fingerprint 或 journal gap 錯誤都會原子拒絕。
- 每個 tick 先持久化再送 frame/result。開發預設使用記憶體 store；同時設定 `REDIS_URL` 與 `DATABASE_URL` 時改用 Redis fencing lease＋PostgreSQL durable record。真實雙客戶端 socket smoke 已證明斷線後 tick 延續且舊命令不重複扣資源。

- v0.15.0／TASK-019 新增 `village-siege-network/1` 精確協定／規則握手；舊版、缺漏或不相容客戶端會在大廳與私有戰局開始前遭拒。瀏覽器提交 `commandId`、每玩家序號、最後可見伺服器 tick 與遊戲命令，可信的比賽／玩家身分仍由伺服器席位注入。
- 每位玩家具備 16 筆有界重排窗與 512 筆完成結果快取：提早抵達的序號等待缺口，相同 pending 命令不重排，相同 completed 命令重播原結果；相同 ID 不同內容或相同序號不同 ID 均拒絕且不改世界。命令結果只走一條專用通道。
- 10 Hz 傳輸改為玩家過濾後的 delta；tick 0、每 50 tick（5 秒）、手動 resync 與終局傳完整 Snapshot。Delta 必須銜接精確 base tick／可見 checksum，客戶端在候選副本完整重建並驗證後才原子換入。每 20 tick（2 秒）的 canonical hash 只留在伺服器，不成為霧外狀態 oracle；大廳也不再公開種子。
- 本機雙客戶端 smoke 已實際刻意漏掉一個 delta、偵測斷鏈、取得完整 Snapshot 恢復，並重送同一個產兵命令驗證只扣一次 50 糧；同時通過 212 個 shared、38 個 client、18 個 server 測試。TASK-020～022 的耐久重連、線上 Phaser 戰場與完整延遲／惡意 E2E 尚未完成，因此仍不宣稱可玩線上對戰。
- v0.14.0／TASK-018 將六碼房大廳與私有權威戰局拆成兩種 Colyseus room。大廳只管理 2～5 人名單、準備與一次性席位交接；伺服器先替全員預約並鎖滿戰局席位，瀏覽器只能消費自己的 Colyseus reservation。戰局由私有 `MatchAuthority` 每 100 ms 執行一次共享模擬，瀏覽器不再能指定 `matchId` 或 `playerId`。
- 每位玩家只收到自己的 `VisibleSnapshot` 與投影事件；完整 `MatchState`、霧外敵軍、AI 私有狀態、佇列與 PRNG 不進 Colyseus schema。不同玩家相同 sequence 的回執分流，已排入 Tick 的序號不會因後續語意拒絕而回退重用。
- 本機雙客戶端 smoke 驗證大廳／戰局分房、伺服器預約的不透明 SDK 席位、未預約者無法搶位、公開建立戰局遭拒、相同 sequence 回執隔離、偽造單位控制與注入身分欄位遭拒，以及伺服器接受訓練後實際扣除 50 糧。當時 TASK-019～022 仍未完成。
- TASK-017 新增手機可完成的七段互動教學，使用均衡型新手 AI 且不提供資源、視野或傷害作弊。採集、升階、研究、探索、戰鬥與破口只接受玩家先前下達且被規則引擎接受的命令；進度只讀玩家可見 Snapshot 與投影事件，敵方塔自動傷害、開局自動採集、友軍破口、敗北或和局都不能偷算。
- 戰前固定三鍵改為「互動教學／多人連線／開始單機戰役」；工匠指揮塢新增採糧、伐木、採石直接鍵，避免手機上資源與城防命中區相鄰而難以操作。直向阻擋時所有 Canvas 鍵盤代理同步隱藏、停用並釋放焦點，橫向後恢復第一個可用操作。
- TASK-017 於 2026-07-21 通過 Chromium 568×320、667×375、844×390、1280×720、390×844、完整測試與雙重稽核；Codex 與 Grok CLI 最終 P0／P1／P2 均無未結項目。
- v0.13.0 新增帶 `schemaVersion`、`protocolVersion`、`rulesVersion` 的單機存檔 Snapshot、逐 tick 指令日誌與決定論 replay；三層版本必須逐字精確相容，未知、缺漏或不相符版本都會明確拒絕，不做猜測式降級載入。
- 存檔與 replay 保存完整權威 `MatchState`，包含 PRNG、AI 私有規劃狀態、霧外實體、視野、佇列、投射物與勝利狀態。這些檔案是擁有者私有的恢復／稽核資料，不得作為玩家公開 Snapshot 或未授權分享格式。
- 指令日誌依權威 tick 記錄已接受的人類／AI 命令、實際提交的 AI 私有 authority 與固定 tick 推進；重播在原提交點重走命令驗證與世界模擬，並逐操作比對前後狀態 hash。被拒命令不進日誌，但最終 runtime metadata 會保留已消耗的玩家序號、未滿一 tick 的時間餘數與 AI budget，並以獨立 continuation hash 綁定最終狀態。FNV-1a 僅是意外損壞與 desync 診斷訊號，不能證明來源、機密性或抵抗惡意竄改。
- TASK-016 的發布門檻包含固定種子 10,000 tick 重播：相同 Snapshot 與 journal 會重複產生相同 checkpoint 和最終 canonical state hash；完整測試、Codex 獨立稽核與 Grok CLI 補丁複核已於 2026-07-21 通過，P0／P1／P2 均無未結項目。
- v0.12.0 新增可雜湊、可複製、可投影的完整勝利狀態：議事核心重建寬限、全軍殲滅、拓界銅標持續控制與中域計時控制都由固定 10 Hz 共享模擬結算。
- 同一 tick 先批次套用所有合法命令，再一次判定隊伍淘汰與勝利；同刻不同隊伍達成條件會明示和局，多條勝途同刻完成會保留完整觸發原因，不依輸入排列或字典序偷選勝方。
- 投降或淘汰後的單位、守望塔、生產佇列與投射物立即失效；終局事件只發布一次，公開 Snapshot 保留結果、隊伍分數、原因與完成 tick，歷史投降不會覆蓋後續真正的征服原因。
- 單機戰役正式啟用四條勝途。桌面與手機固定 HUD 持續顯示拓界／中域進度與勝／敗／和局；結局沿用既有「再戰／返回」兩個操作，另加入一次完整的螢幕閱讀器播報與鍵盤焦點移轉。
- v0.11.0 將 AI 隨機狀態、決策節奏、敵情記憶、反制鎖定、修理目標、撤退／集結階段與進攻波次納入可雜湊、可複製的權威 `MatchState`；Runtime 不再另存前端 AI 序號或控制器閉包。
- 新增純函式策略 reducer：五種 AI 只使用玩家可見敵軍、已探索格與權威最後目擊，能穩定選擇反制編成、優先修理城門／主城／塔／牆、評估劣勢撤退、保留守軍、集結增援並依冷卻發動後續攻勢。
- AI 空間模型現在辨識三張村莊各自地形、開門、城牆、殘骸、可見建造格與可達卸貨點；採集、建造與集結只送出合法子集合，避免以遭拒命令探測戰爭迷霧。
- 敵方策略階段只透過視野過濾後的短暫戰術提示呈現；提示不包含兵力、路徑、目標座標、波次編號或私人 AI 記憶，且不增加手機七格指揮塢按鈕。
- v0.10.0 新增松林堡、河谷鎮與高地寨三張原創 18×16 地形配置；河道、岩脊、林帶、道路與保留通道由共享資料驅動，兩人戰役使用完整城牆、城門、雙塔、經濟建築與三名持續工作的平民開局。
- 沼牙獸、燼翼獵獸與根背巨像正式進入權威模擬：各有索敵規則、範圍技能、狀態效果、冷卻與被動，討伐貢獻依隊伍結算並授予限時採集、移速或攻擊增益。
- 修正延遲投射物與燃燒在來源死亡後遺失歸屬、致命同 tick 外怪事件投影漏失、隱藏攻擊者隊伍洩漏，以及三至五人相容開局把主城、單位或資源放到非法地形的問題。
- 戰役畫面新增六向外怪動作、工匠採集／搬運／施工姿勢、死亡退場、破口煙塵、討伐獎勵與增益倒數；手機遊戲區阻擋 HUD 穿透，直接點選命中區加大並維持固定七格指揮塢。
- v0.9.0 新增原創樹脂石籠牆、測界雙葉門與城寨期拓界銅標；資料驅動定義涵蓋成本、護甲、施工時間、朝向、旋轉占地、移動阻擋與破壞後殘骸。
- 新增擁有者限定的 `setGateState` 命令與 `gateStateChanged` 事件。完成且存活的雙葉門可開關；關閉前會拒絕門格內仍有單位的命令，且遭拒時不改狀態。
- 建造占位與移動阻擋改成兩套明確拓撲：開啟城門可通行但不能在門上重疊建造；城防摧毀後產生可通行、暫時阻擋放置並於固定 200 tick 消散的決定論破口殘骸。
- 單位遇到關門或未完成城防不會丟失原移動命令；門開啟或城防被突破後會按原目標重新尋路。路徑規劃只採用盟友實況、可見敵軍與最後目擊資料，實際碰撞也不會藉由隱藏城門狀態洩漏霧外拓撲。
- 施工生命值依每 tick 新增的施工上限成長，戰鬥傷害不再在完工瞬間被補滿；建築跨越健康、受損、危急與摧毀區間時會發布可見狀態更新。
- Phaser 新增不對稱雙葉門、樹脂綁木石籠、氧化銅拓界標與破口殘骸的原創程式繪製美術。建造選單擴為四頁；放置旋轉與城門控制皆維持固定七格手機指揮塢。
- v0.8.0 新增逐玩家可見／已探索格網、隊伍共享視野、敵方建築最後目擊與固定 RLE 探索資料；完整狀態仍只存在權威模擬，玩家 Snapshot 不序列化霧外即時資料。
- 可見 Snapshot 會遮蔽隱藏敵軍、投射物來源、鎖定目標與未探索目標點；玩家事件投影只保留可證明可見的戰鬥事件及自己的經濟／生產事件，AI 記憶改由相同權威迷霧狀態提供。
