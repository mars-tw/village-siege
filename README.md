# Village Siege／村莊攻防

Village Siege 是一個 MIT 授權的瀏覽器即時戰略遊戲原型。它採用原創的中世紀等角視角美術、程式繪製素材與自有介面；節奏參考經典 RTS，但不包含《世紀帝國 II》的名稱、素材、音效或介面複製品。

## 立即遊玩

**[開啟公開單機版](https://mars-tw.github.io/village-siege/)**

公開網站由 GitHub Pages 自動建置。單機戰役可直接遊玩；多人功能目前仍是房間與連線架構原型，尚未提供完整同步戰場。

原始碼與版本紀錄位於 **[mars-tw/village-siege](https://github.com/mars-tw/village-siege)**。v2 將軍隊生產拆成五座專屬軍事工坊，並加入多格建築占地、單位繞路及手機三頁建造選單；完整改動見 [CHANGELOG](CHANGELOG.md)。

第一次接觸專案請先閱讀 **[繁體中文新手指南](docs/BEGINNER_GUIDE.zh-TW.md)**；內容包含五分鐘啟動、第一場戰役、完整操作、多人房間與常見問題。

目前已核准的開源原型里程碑包含：

- 戰前選單提供三個村莊：松林堡、河谷鎮、高地寨。
- 戰前選單提供五種 AI 性格：進攻、守備、發展、平衡、襲擾；AI 與玩家共用採集、施工、人口、訓練與攻擊規則，會完成「經濟→兵營→產兵→偵查／進攻」循環。
- 18×16、2:1 等角村莊戰場；每方從議事堂、三名工匠與木材、糧食、石礦開始，所有資源與人口都由 10 Hz 決定論模擬管理。
- 可建造拓荒家屋、木作營、糧秣所、邊軍兵營、守望塔、射箭庭、星火院、火器坊、獠騎圈與攻城棚。v3 依聚落階段解鎖：拓荒期先建立基礎經濟與兵營，城寨期開放守望塔、射箭庭與獠騎圈，工藝期再開放星火院、火器坊與攻城棚；未解鎖項目會保留在選單並標明前置。家屋增加人口；主城接收全部材料，木作營與糧秣所分別作為木材、糧食的近端卸貨點；守望塔自動防禦。
- 軍隊依建築分工：兵營訓練戰士與持盾槍衛；射箭庭訓練弓手；星火院訓練法師；火器坊訓練火銃兵；獠騎圈訓練野豬斥候；攻城棚訓練重弩攻城組。成本、人口、時間與最多五格佇列都由共享規則驗證。
- 建築具有 1×1、2×1、2×2 或 3×2 的真實格位占地；放置時會檢查整片基地，單位移動則會繞過已占用格，而不是穿越建築中心。
- 七種軍事單位接入專案原創 4×6 透明動作表；待機、走路、攻擊、施法、受擊、死亡使用不同逐格姿勢，缺少正式素材會阻止開局，不會退回程序人偶。
- 建築具有原創「邊境營造盤」美術、施工鷹架、完成狀態、受損裂紋、生命與訓練進度；工匠最多攜帶 12 單位材料，必須送達合法卸貨點才會入帳。木材與石礦會耗盡，糧田耗盡後會休耕並在固定 tick 復育。
- 角色動作表採戰役場景 lazy-loading：主選單不先下載整套戰鬥素材，進入單人戰役時才載入必要圖集並顯示載入狀態。
- 摧毀敵方議事堂會啟動 60 秒重建寬限；期限內未重建即以征服結束戰役。
- Colyseus 權威伺服器大廳：2–4 位玩家、六碼房號、準備／開始、10 Hz 權威 tick。
- 60 秒斷線重連窗口，以及房主、訊息大小、序號與單位擁有權的基本驗證。

## 環境需求

- Node.js 22.12 或更新版本。
- npm 11 或相容版本。
- 支援 WebGL 的現代瀏覽器。

## 本機啟動

先在專案根目錄安裝鎖定版本：

```powershell
npm ci
```

開啟第一個終端機啟動多人伺服器：

```powershell
npm run dev:server
```

伺服器預設監聽 `http://localhost:2567`。再於第二個終端機啟動遊戲：

```powershell
npm run dev:client
```

開啟 Vite 顯示的網址即可選擇單人模式，或在多人模式建立房間。第二個瀏覽器／無痕視窗可輸入同一個六碼房號加入。

單機戰役操作：

- 點我方工匠或建築選取；`Shift` 可加選。點空地讓已選單位移動，點敵人進攻。
- 選取工匠後點木材、糧食或石礦即可持續採集；滿載後會自動送回合法卸貨點，也可按「卸糧／卸木／卸石」（混合載貨時為「卸全部」）或直接點主城、木作營、糧秣所手動送貨。按 `B` 或底部「建造」，選建築後再點有效空地放置。
- 選取議事堂可訓練工匠；選取已完成的對應軍事建築，才可把該建築負責的兵種加入訓練佇列。
- 拖曳空地平移鏡頭；`WASD` 平移、滾輪縮放、`P` 暫停、`R` 重開、`Esc` 取消建造或返回戰前會議。
- 主城、人口、成本、採集、施工、訓練、傷害與勝負皆由同一份共享狀態決定，畫面不會自行增加假資源。

手機與觸控裝置操作（568×320、667×375、844×390、390×844 與 1366×1024 已驗證）：

- 點我軍選取、點地移動、點資源採集、點敵軍攻擊，拖曳空地可平移鏡頭；工匠滿載會自動卸貨，指令列也提供手動卸貨。
- 底部是依目前選取物切換的單列 Canvas 指揮塢；十種可建建築分成三頁，以「下一頁／回首頁」循環切換，建造、產兵與系統操作不會疊出第二層視窗。
- 遊戲會偵測觸控裝置、可視瀏覽器高度與橫直方向，以 `EXPAND` 模式填滿可用畫面；直向時會提示將手機橫放。
- 手機點「開始單機戰役」時會在該次使用者操作中要求全螢幕；也可用選村畫面的「全螢幕」或戰場右上角軍令牌切換。若瀏覽器不支援 Fullscreen API，則改用最大可視滿版並顯示提示。
- 568×320 的主命令觸控區實測至少約 52×52 CSS px；上下資訊列與指揮塢均留在遊戲畫布內且不重疊。
- Canvas 內提供拉近、拉遠、暫停、選取工匠／軍隊與全螢幕；每個可見 Canvas 指令都有鍵盤／螢幕閱讀器代理。

若伺服器不在預設位置，啟動或建置客戶端前設定：

```powershell
$env:VITE_COLYSEUS_URL = "http://127.0.0.1:2567"
npm run dev:client
```

## 驗證與多人 smoke test

完整型別檢查、共享模擬測試與正式建置：

```powershell
npm run verify
```

可重跑的本機多人測試會自行建置、啟動獨立伺服器、建立兩個客戶端，最後關閉連線與伺服器：

```powershell
npm run smoke:multiplayer:local
```

它驗證以下行為：

- 房主建房、第二位玩家依房號加入同一房間。
- 非房主無法開始；玩家未全數準備時房主也無法開始。
- 非法 ready payload 遭拒。
- 斷線後可用原 session 自動重連。
- 兩位玩家準備後由房主開始，雙方收到相同的 playing 階段與持續增加的伺服器 tick。

若已另行啟動伺服器，可改用：

```powershell
npm run smoke:multiplayer
```

以 `COLYSEUS_URL` 指定非預設測試端點；自啟測試則可用 `SMOKE_PORT` 改變預設的 `26567` 連接埠。

## 建置與發布

客戶端建置時必須把公開的 Colyseus HTTPS 端點寫入 `VITE_COLYSEUS_URL`，瀏覽器 SDK 會使用對應的安全 WebSocket：

```powershell
$env:VITE_COLYSEUS_URL = "https://game.example.com"
npm run build
```

- 將 `apps/client/dist/` 發布至靜態網站或 CDN。
- 以 `PORT` 指定伺服器連接埠，再執行 `npm run start:server`。
- 公開部署時應在反向代理終止 TLS、使用 `wss`、設定來源限制、速率限制、監控與健康檢查。

### 自有設計與完整開源

本專案的「私有化」指 **改造成 Village Siege 自己的品牌、玩法、美術與世界觀**，不是把程式碼鎖進私有倉庫。客戶端、伺服器、共享規則、美術來源、部署範例與 infrastructure 範本都以 MIT 完整開放；只有密碼、Token、私鑰等實際憑證不得提交。完整的 server-authoritative 遷移與全開源部署架構見 [v3 authoritative RTS architecture](docs/architecture/authoritative-rts-v3.md)。

## 已知限制與發布風險

- v3 開發分支已加入拓荒期、城寨期、工藝期三階段進程、前置建築、資料驅動解鎖、資源搬運／卸貨、有限木石與可復育糧田；仍未完成研究科技樹、修理、戰爭迷霧與牆門。
- 建築占地與靜態阻擋繞路已由共享規則處理，但尚未加入大部隊隊形、移動單位間的動態避讓與長距離路徑快取；大量單位擠進狹窄入口時仍可能排隊。
- 三村目前共用同一張戰場與建築配置；松林堡有 3% 採集加成、河谷鎮有 3% 移速加成、高地寨有 4% 守望塔護甲，且都由共享模擬計算，但尚未各自擁有獨立科技樹。
- 工匠暫時共用戰士動作表並加上工具標記；七種軍事單位皆有正式動作表，但工匠專屬採集／施工影格仍待製作。
- 現階段多人模式完成房間、準備、開始、重連及權威 tick；戰場的完整共享模擬與客戶端指令仍未接到線上狀態，因此尚不是完整的線上對戰版本。
- 現有角色動作表完成六動作、每動作四個獨立影格；目前邏輯朝向仍共用右向母版並以左右鏡像呈現，尚未完成六方向各自獨立繪製的 multiatlas。
- 房間與比賽狀態僅存於單一 Node.js 程序記憶體，沒有資料庫、跨程序 presence、伺服器遷移或災難復原。
- 房號由客戶端提出，伺服器會正規化與篩選，但尚未提供全域唯一性保證或邀請權限。
- 目前沒有帳號驗證、完整 anti-cheat、每玩家速率限制、管理員工具與觀戰隱私控制；不可直接視為網際網路公開服務的安全完成品。
- 戰役素材已延後到進入場景時載入；JavaScript 場景程式與多人功能仍可在後續版本繼續拆分 chunk，以降低首次開啟的傳輸量。
- 2026-07-19 執行 `npm audit --omit=dev` 為 0 vulnerabilities。伺服器直接使用 `@colyseus/core`、schema、WebSocket transport 與 Express，沒有引入未使用的 Colyseus auth/playground meta-package 相依鏈；公開發布前仍應在 CI 持續執行 audit 與版本審查。
