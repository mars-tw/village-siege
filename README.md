# Village Siege／村莊攻防

Village Siege 是一個 MIT 授權的瀏覽器即時戰略遊戲原型。它採用原創的中世紀等角視角美術、程式繪製素材與自有介面；節奏參考經典 RTS，但不包含《世紀帝國 II》的名稱、素材、音效或介面複製品。

## 立即遊玩

**[開啟公開單機版](https://mars-tw.github.io/village-siege/)**

公開網站由 GitHub Pages 自動建置。單機戰役可直接遊玩；多人功能目前仍是房間與連線架構原型，尚未提供完整同步戰場。

原始碼與版本紀錄位於 **[mars-tw/village-siege](https://github.com/mars-tw/village-siege)**。v0.13 為單機權威狀態加入版本化存檔、逐 tick 指令日誌與決定論重播；v0.12 已加入四條決定論勝途，v0.11 的五種 AI 策略規劃器則負責偵察記憶、穩定反制、城防維修、撤退集結與多波進攻。完整改動見 [CHANGELOG](CHANGELOG.md)。

第一次接觸專案請先閱讀 **[繁體中文新手指南](docs/BEGINNER_GUIDE.zh-TW.md)**；內容包含五分鐘啟動、第一場戰役、完整操作、多人房間與常見問題。

目前已核准的開源原型里程碑包含：

- 戰前選單提供三個村莊：松林堡、河谷鎮、高地寨。
- 戰前選單提供五種 AI 性格：進攻、守備、發展、平衡、襲擾；AI 與玩家共用採集、施工、人口、訓練與攻擊規則，會完成「經濟→兵營→產兵→偵查／進攻」循環。
- 18×16、2:1 等角村莊戰場；每方從議事堂、三名工匠與木材、糧食、石礦開始，所有資源與人口都由 10 Hz 決定論模擬管理。
- 可建造拓荒家屋、木作營、糧秣所、邊軍兵營、守望塔、射箭庭、星火院、火器坊、獠騎圈與攻城棚。v3 依聚落階段解鎖：拓荒期先建立基礎經濟與兵營，城寨期開放守望塔、射箭庭與獠騎圈，工藝期再開放星火院、火器坊與攻城棚；未解鎖項目會保留在選單並標明前置。家屋增加人口；主城接收全部材料，木作營與糧秣所分別作為木材、糧食的近端卸貨點；守望塔自動防禦。
- 軍隊依建築分工：兵營訓練戰士與盾牌手；射箭庭訓練弓箭手；星火院訓練法師；火器坊訓練火槍兵；獠騎圈訓練野豬騎士；攻城棚訓練重弩攻手。七兵種的生命、護甲、傷害、射程、移速、成本、克制與主動技能都由同一份共享規則衍生。
- v0.7.0 將七兵種正式接入單機權威戰鬥：近戰蓄力、遠程飛行、地形遮蔽、護甲與克制、技能冷卻、職業被動與狀態效果都按固定 tick 結算；箭雨／法印在命中時掃描區域，破城巨矢逐 tick 沿路碰撞。玩家、箭塔與 AI 共用隊伍敵我判定，不會攻擊同隊單位。
- v0.8.0 加入共享決定論戰爭迷霧：每位玩家分別保存目前可見與永久探索格，隊友共享視野；移動敵軍離開視野即隱藏，敵方建築只留下不可互動、且不會偷更新血量的半透明最後目擊。玩家專屬 Snapshot 與單 tick 事件投影會移除霧外實體、命中名單、投射物目標座標與傷害來源，AI 也只使用同一份權威視野與記憶。
- v0.9.0 加入原創築城垂直切片：樹脂石籠牆、可旋轉的測界雙葉門與城寨期拓界銅標皆有共享成本、護甲、施工、朝向、占地及生命狀態。關門阻擋移動、開門放行但仍保留建造占位；摧毀後會生成暫時阻擋建造但可供部隊穿越的決定論破口殘骸。門的開關、受損狀態與最後目擊會經玩家專屬可見資料投影，不從戰爭迷霧洩漏即時拓撲。
- v0.10.0 將松林堡、河谷鎮與高地寨改為各自資料驅動的連續地形與保留通道；標準雙人戰役從完整城牆、城門、雙塔、生產／經濟建築、資源點與三名工作平民開始。三至五人相容開局也會決定論尋找合法地形，不把建築或資源塞進河道、岩地或保留通道。
- v0.11.0 把 AI 種子、敵情記憶、反制鎖定、修理目標、撤退點與波次成員納入可雜湊、可複製的權威比賽狀態。五種性格會依目前視野與合法最後目擊偵察、守牆、修理、撤退、集結並分波進攻，所有命令仍走玩家共用驗證器。
- v0.12.0 加入議事核心征服、全軍殲滅、拓界銅標持續控制與中域計時控制。四條勝途共用固定 tick、同刻和局與一次性結果事件；手機固定畫面會持續顯示進度，v0.13 終局列另提供完整重播下載。
- v0.13.0 定義含 `schemaVersion`、`protocolVersion`、`rulesVersion` 的存檔、逐 tick 操作日誌與重播格式，三層版本都必須精確相容才可載入。重播會重新驗證已接受命令、在原提交點還原 AI 私有 authority，並逐步核對固定 tick 的狀態；最終狀態與續玩序號／子 tick 餘數／AI budget 另由 continuation hash 綁定。因此檔案含霧外敵情與 AI 記憶，只適合擁有者本機保存，不是可公開傳給玩家的 `VisibleSnapshot`。FNV 雜湊只用來偵測意外損壞與決定論分歧，不提供加密或防竄改保證。
- 戰前畫面提供可由觸控完成的七段互動教學：採集卸貨、聚落升階、科技研究、未知地形探索、實際戰鬥、指定城防破口與合法勝利。教學使用新手 AI，但不贈送資源、不開黑霧、不改傷害；只有玩家接受過的採集、探索、攻擊與破口命令才能推進對應目標。
- 三種原創外怪都由共享模擬處理：沼牙獸伏擊緩速、燼翼獵獸俯衝後排、根背巨像重擊建築並在低生命狂怒。討伐依傷害貢獻把資源與限時增益發給隊伍；客戶端只顯示權威結果、技能動作、獎勵提示與倒數。
- 七項原創科技涵蓋採集、耐久、移速與兵種攻擊；科技與產兵共用建築生產線，檢查聚落階段、前置科技、資源與全玩家唯一完成狀態。五種 AI 各有不同研究優先順序，完成效果由共享決定論模擬套用。
- 每項訓練與研究都有穩定工作 ID；玩家可在固定七格佇列頁先選取、再確認取消。等待工作全額退款，已開工項目依剩餘進度退款，延遲操作不會誤刪已移位的下一項。
- 可產兵建築能設定原創銅色集結旗；新兵從合法周邊格生成後前往旗點，若旗點後來被封鎖則新兵保持待命，生產佇列仍會繼續。
- 建築具有 1×1、2×1、2×2 或 3×2 的真實格位占地；放置時會檢查整片基地，單位移動則會繞過已占用格，而不是穿越建築中心。
- 七種軍事單位接入專案原創 4×6 透明動作表；待機、走路、攻擊、施法、受擊、死亡使用不同逐格姿勢，缺少正式素材會阻止開局，不會退回程序人偶。
- 建築具有原創「邊境營造盤」美術、施工鷹架、完成狀態、受損裂紋、生命與訓練進度；工匠最多攜帶 12 單位材料，必須送達合法卸貨點才會入帳。木材與石礦會耗盡，糧田耗盡後會休耕並在固定 tick 復育。
- 角色動作表採戰役場景 lazy-loading：主選單不先下載整套戰鬥素材，進入單人戰役時才載入必要圖集並顯示載入狀態。
- 摧毀敵方議事核心會啟動 60 秒重建寬限；也可藉由殲滅全部有效戰力、持有一座拓界銅標 90 秒，或在開局 60 秒後累積控制中域 120 秒獲勝。
- Colyseus 伺服器大廳原型：2–4 位玩家、六碼房號、準備／開始與 10 Hz server-owned lobby clock；此 clock 尚未執行共享戰場模擬。
- 60 秒斷線重連窗口，以及房主、訊息大小、序號與單位擁有權的基本驗證。

## 環境需求

- Node.js 22.12 或更新版本。
- npm 11 或相容版本。
- 支援 WebGL 的現代瀏覽器。

## 本機啟動

先在專案根目錄安裝鎖定版本：

```powershell
npm ci
```

開啟第一個終端機啟動多人伺服器：

```powershell
npm run dev:server
```

伺服器預設監聽 `http://localhost:2567`。再於第二個終端機啟動遊戲：

```powershell
npm run dev:client
```

開啟 Vite 顯示的網址即可選擇單人模式，或在多人模式建立房間。第二個瀏覽器／無痕視窗可輸入同一個六碼房號加入。

單機戰役操作：

- 點我方工匠或建築選取；`Shift` 可加選。點空地讓已選單位移動，點敵人進攻。
- 選取工匠後可直接按「採糧／伐木／採石」，或點木材、糧食、石礦持續採集；滿載後會自動送回合法卸貨點，也可按「卸糧／卸木／卸石」（混合載貨時為「卸全部」）或直接點主城、木作營、糧秣所手動送貨。按 `B` 或底部「建造」，選建築後再點有效空地放置。
- 測界雙葉門放置時可用固定指揮塢切換北東／南東軸；完工後點選城門即可開啟或關閉。城門格內有單位時，關門命令會被權威規則拒絕，避免把部隊夾在門內。
- 選取議事堂可訓練工匠；選取已完成的對應軍事建築，才可把該建築負責的兵種加入訓練佇列。建築底部第五格開啟佇列，第六格進入集結選點；取消工作必須再按一次確認。
- 純軍隊選取時，固定七格依序提供攻擊移動、雙點巡邏、編隊、姿態、兵種技能、停止與系統。攻擊移動會沿途尋敵並在交戰後續行；編隊提供橫列、楔形、方陣，姿態提供主動、守備、堅守。選取工匠則可點「修復」後指定受損友方建築。
- 拖曳空地平移鏡頭；`WASD` 平移、滾輪縮放、`P` 暫停、`R` 重開。`Esc` 會依序取消戰術選點、從取消確認返回佇列、從佇列／集結選點返回建築主指令、取消建造，最後才返回戰前會議。
- 主城、人口、成本、採集、施工、訓練、傷害與勝負皆由同一份共享狀態決定，畫面不會自行增加假資源。

手機與觸控裝置操作（568×320、667×375、844×390、390×844 與 1366×1024 已驗證）：

- 點我軍選取、點地移動、點資源採集、點敵軍攻擊，拖曳空地可平移鏡頭；工匠列另有「採糧／伐木／採石」直接鍵，解決小螢幕上資源與建築命中區相鄰的問題。工匠滿載會自動卸貨，指令列也提供手動卸貨。
- 底部是依目前選取物切換的單列 Canvas 指揮塢；十三種可建建築分成四頁，以「下一頁／回首頁」循環切換。建築旋轉、城門開關、軍隊的攻擊移動、巡邏、編隊、姿態、技能、停止、系統，以及生產佇列、取消確認、集結與修復選點，都在同一排七格內完成，不會疊出第二層視窗。
- 遊戲會偵測觸控裝置、可視瀏覽器高度與橫直方向，以 `EXPAND` 模式填滿可用畫面；直向時會提示將手機橫放。
- 手機點「互動教學」或「開始單機戰役」時會在該次使用者操作中要求全螢幕；戰場內可由 **系統** 切換全螢幕。若瀏覽器不支援 Fullscreen API，則改用最大可視滿版並顯示提示。
- 568×320 的主命令觸控區實測至少約 52×52 CSS px；上下資訊列與指揮塢均留在遊戲畫布內且不重疊。
- Canvas 內提供拉近、拉遠、暫停、選取工匠／軍隊與全螢幕；每個可見 Canvas 指令都有鍵盤／螢幕閱讀器代理。

若伺服器不在預設位置，啟動或建置客戶端前設定：

```powershell
$env:VITE_COLYSEUS_URL = "http://127.0.0.1:2567"
npm run dev:client
```

## 驗證與多人 smoke test

完整型別檢查、共享模擬測試與正式建置：

```powershell
npm run verify
```

可重跑的本機多人測試會自行建置、啟動獨立伺服器、建立兩個客戶端，最後關閉連線與伺服器：

```powershell
npm run smoke:multiplayer:local
```

它驗證以下行為：

- 房主建房、第二位玩家依房號加入同一房間。
- 非房主無法開始；玩家未全數準備時房主也無法開始。
- 非法 ready payload 遭拒。
- 斷線後可用原 session 自動重連。
- 兩位玩家準備後由房主開始，雙方收到相同的 playing 階段與持續增加的伺服器 tick。

若已另行啟動伺服器，可改用：

```powershell
npm run smoke:multiplayer
```

以 `COLYSEUS_URL` 指定非預設測試端點；自啟測試則可用 `SMOKE_PORT` 改變預設的 `26567` 連接埠。

## 建置與發布

客戶端建置時必須把公開的 Colyseus HTTPS 端點寫入 `VITE_COLYSEUS_URL`，瀏覽器 SDK 會使用對應的安全 WebSocket：

```powershell
$env:VITE_COLYSEUS_URL = "https://game.example.com"
npm run build
```

- 將 `apps/client/dist/` 發布至靜態網站或 CDN。
- 以 `PORT` 指定伺服器連接埠，再執行 `npm run start:server`。
- 公開部署時應在反向代理終止 TLS、使用 `wss`、設定來源限制、速率限制、監控與健康檢查。

### 自有設計與完整開源

本專案的「私有化」指 **改造成 Village Siege 自己的品牌、玩法、美術與世界觀**，不是把程式碼鎖進私有倉庫。客戶端、伺服器、共享規則、美術來源、部署範例與 infrastructure 範本都以 MIT 完整開放；只有密碼、Token、私鑰等實際憑證不得提交。完整的 server-authoritative 遷移與全開源部署架構見 [v3 authoritative RTS architecture](docs/architecture/authoritative-rts-v3.md)。

## 已知限制與發布風險

- v3 開發分支已加入三階段聚落、七項科技、資源搬運、戰爭迷霧、築城與破口、三張強化村莊地形、工作平民、權威野外怪物、策略 AI、四條決定論勝途、已通過 TASK-016 驗證的 v0.13 存檔／重播契約，以及已通過 TASK-017 手機、無障礙、Codex 與 Grok CLI 門檻的七段互動教學。全戰場權威多人同步仍尚待完成。
- 建築占地與靜態阻擋繞路已由共享規則處理，也已加入橫列、楔形與方陣的穩定格位編隊；尚未加入移動單位間的動態避讓與長距離路徑快取，大量單位擠進狹窄入口時仍可能排隊。
- 三村目前共用同一張戰場與建築配置；松林堡有 3% 採集加成、河谷鎮有 3% 移速加成、高地寨有 4% 守望塔護甲，且都由共享模擬計算，但尚未各自擁有獨立科技樹。
- 工匠暫時共用戰士動作表並加上工具標記；七種軍事單位皆有正式動作表，但工匠專屬採集／施工影格仍待製作。
- 現階段多人模式完成房間、準備、開始、重連及 server-owned lobby clock；戰場的完整共享模擬與客戶端指令仍未接到線上狀態，因此尚不是完整的線上對戰版本。
- 現有角色動作表完成六動作、每動作四個獨立影格；目前邏輯朝向仍共用右向母版並以左右鏡像呈現，尚未完成六方向各自獨立繪製的 multiatlas。
- 房間與比賽狀態僅存於單一 Node.js 程序記憶體，沒有資料庫、跨程序 presence、伺服器遷移或災難復原。
- 房號由客戶端提出，伺服器會正規化與篩選，但尚未提供全域唯一性保證或邀請權限。
- 目前沒有帳號驗證、完整 anti-cheat、每玩家速率限制、管理員工具與觀戰隱私控制；不可直接視為網際網路公開服務的安全完成品。
